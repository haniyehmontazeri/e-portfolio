MHI Driver Drowsiness Detection System



We had seminar at Eastern Meditterranean University which was about IOT components and sensors.In this seminar the speaker has talked about Arduino board and different sensors and The IDE of Arduino. Therefore after this seminar by doing some research I developed MHI Driver Drowsiness Detection system that can detect driver drowsiness and do some tasks.

The face indicates information about different modes. For example, When the driver is drowsy, the frequency of blinking is lower and compared to the normal mode, the blinking time is longer. Driver drowsiness and fatigue are the main reason of the road accidents. Various studies have suggested that around 20% of all road accidents are fatigue- related, up to 50% on certain roads which is more than the percentage of those in drink-driving. In this paper, we designed a system called MHI Driver Drowsiness Detection System to reduce the number of accidents and increase the level of car safety. In this system, we developed an algorithm by the use of OpenCV libraries and image processing in order to analysis the eye state based on 6 landmarks. It detects driver’s face and eyes to measure EAR (Eye Aspect Ratio) of the eyes which determines whether the eyes are open or closed, and according to the eye state, warns the drowsiness of driver. The design of this device is such that it can be installed on any vehicle and is used in a portable way.



SOFTWARE: Blinking is a semi-autonomic rapid function of closing eyelid. In this project we use face-recognition library and facial landmark detector in two different sections. In the first section which is called face detection, try to recognize the driver’s face in a rectangle in order to limit the area that going to work with. Then, after detecting the face, it’s time to point inside the rectangle especially the eyes. Eye Aspect Ratio is a method determines the close or open state of the eyes. In this method, it uses different landmarks to distinguish the state of the eyes. The result will yield whether the eye is open or not, because the eye aspect ratio will approach to minimum EAR value during a blink.



HARDWARE: The design of the vision system board is made with Altium software. We used Raspberry Pi board for processing data, and Servo motors to move cameras in order to set them in a desire location.
In the final step, we design a product to match the camera, software, and hardware for users in order to make it easy for them to use this system. It has 3.5-inch LCD to display GUI and warnings, connected to a Raspberry Pi 4 model B, which includes 2.4 GHz and 5.0 GHz IEEE 802.11ac wireless, Bluetooth 5.0, Ble Gigabit Ethernet that enable it to connect to other devices, 2 USB 3.0 ports,2 USB 2.0 ports for connecting camera, and Micro-SD card slot for loading operating system and data storage.
